INET NOPS UAT cURL / Body / Path Details

Prepared for INET support | Date: 2026-06-26 | Project: SST iPOS

เอกสารนี้สามารถส่งให้ INET ได้ โดยปิดบังค่า Merchant Key และ Access Token จริงทั้งหมด

Current Issue

ระบบ POS เรียก INET UAT ได้ถึงขั้น OAuth และ Access Token แต่ขั้นสุดท้าย Create Payment ถูกตอบกลับเป็น HTTP 403 พร้อม HTML page แทน JSON จึงยังไม่สามารถสร้าง QR ได้ 

inet_nops_create_payment_http_403:html_response

รบกวน INET ช่วยตรวจสอบ:

  • Merchant Key UAT นี้เปิดสิทธิ์ create-payment QR แล้วหรือยัง
  • ต้อง whitelist IP/domain จาก Vercel หรือไม่
  • Step 3 ต้องใช้ POST JSON body { "accessToken": "..." } ถูกต้องไหม หรือต้องใช้ method/header/body แบบอื่น
  • apUrl ต้องลงทะเบียนหรือ approve ก่อน create-payment หรือไม่

Public POS URLs

apUrl / Result page:

https://sstipos-ten.vercel.app/payment/inet/result

Server-to-server callback URL:

https://sstipos-ten.vercel.app/api/payments/inet/callback

Step 1: OAuth

Method: POST

Path / URL:

https://new-ops-poc.inet.co.th/uat/oauth/api/v1/oauth-token

Body:

{
  "key": "<INET_UAT_MERCHANT_KEY>",
  "orderId": "SST260626110606ABCDEF123456789"
}

cURL:

curl -i -X POST "https://new-ops-poc.inet.co.th/uat/oauth/api/v1/oauth-token" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  --data '{
    "key": "<INET_UAT_MERCHANT_KEY>",
    "orderId": "SST260626110606ABCDEF123456789"
  }'

Expected response:

{
  "code": 201,
  "token": "<OAUTH_TOKEN>"
}

Step 2: Access Token

Method: POST

Path / URL:

https://new-ops-poc.inet.co.th/uat/api/v1/sandbox/payment-transactions/access-token

Body:

{
  "key": "<INET_UAT_MERCHANT_KEY>",
  "orderId": "SST260626110606ABCDEF123456789",
  "orderDesc": "POS TEST-BILL",
  "amount": 125,
  "apUrl": "https://sstipos-ten.vercel.app/payment/inet/result",
  "payType": "QR",
  "regRef": ""
}

cURL:

curl -i -X POST "https://new-ops-poc.inet.co.th/uat/api/v1/sandbox/payment-transactions/access-token" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <OAUTH_TOKEN>" \
  --data '{
    "key": "<INET_UAT_MERCHANT_KEY>",
    "orderId": "SST260626110606ABCDEF123456789",
    "orderDesc": "POS TEST-BILL",
    "amount": 125,
    "apUrl": "https://sstipos-ten.vercel.app/payment/inet/result",
    "payType": "QR",
    "regRef": ""
  }'

Expected response:

{
  "code": 201,
  "accessToken": "<PAYMENT_ACCESS_TOKEN>",
  "link": "<CREATE_PAYMENT_LINK>"
}

Step 3: Create Payment

Method currently used by POS: POST

Path / URL:

<CREATE_PAYMENT_LINK returned from Step 2>

Body:

{
  "accessToken": "<PAYMENT_ACCESS_TOKEN>"
}

cURL:

curl -i -X POST "<CREATE_PAYMENT_LINK>" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  --data '{
    "accessToken": "<PAYMENT_ACCESS_TOKEN>"
  }'

Expected response:

{
  "code": 200,
  "qrCode": "<BASE64_QR_IMAGE>"
}
Actual current result: HTTP 403, Content-Type: text/html

POS Order ID Format

ระบบ POS สร้าง provider order ID ความยาวไม่เกิน 30 ตัวอักษร:

SST + YYMMDDHHMMSS + 15 random uppercase hex characters

ตัวอย่าง:

SST260626110606ABCDEF123456789

Notes

  • Merchant Key เป็น server-only และไม่ส่งไป browser
  • ปุ่มทดสอบ UAT ในหน้าตั้งค่าทดสอบเฉพาะ OAuth เท่านั้น
  • การสร้าง QR จริงต้องผ่านทั้ง 3 ขั้นตอนด้านบน
  • บิลจะปิดอัตโนมัติหลัง INET ส่ง callback สำเร็จด้วย event=payment_status_change และ detail.response_code=0
